Learn how to install SigNoz on Docker.

## Prerequisites

- A Linux or macOS machine. Microsoft Windows is not officially supported.
- **Docker** and **Docker Compose**: Install <a href="https://docs.docker.com/engine/install/" target="_blank" rel="noopener noreferrer nofollow">Docker Engine</a> and <a href="https://docs.docker.com/compose/install/" target="_blank" rel="noopener noreferrer nofollow">Docker Compose</a> on your machine.
- A minimum of 4GB of memory allocated to Docker.
- For manual install only: <a href="https://desktop.github.com/" target="_blank" rel="noopener noreferrer nofollow">Git client</a> (to clone the repo).

## Installation Methods

- **<a href="https://github.com/SigNoz/foundry" target="_blank" rel="noopener noreferrer nofollow">Foundry</a> (recommended)**: A tool that handles deploying SigNoz for you, so you can get started quickly and easily. You specify what you want, Foundry sets it up.
- **Manual install**: Clone the SigNoz repo and run `docker compose up` yourself.

<Tabs>
<TabItem value="foundry" label="Foundry" default>

<a href="https://github.com/SigNoz/foundry" target="_blank" rel="noopener noreferrer nofollow">Foundry</a> provides <a href="https://github.com/SigNoz/foundry/tree/main/docs/examples/docker/compose" target="_blank" rel="noopener noreferrer nofollow">Docker Compose casting</a> to run SigNoz on Docker. Foundry generates the Compose file and configs, then deploys with Docker.

### Step 1. Install foundryctl

`foundryctl` is the command-line tool you use to deploy SigNoz with Foundry.

Download and extract the binary from <a href="https://github.com/SigNoz/foundry/releases" target="_blank" rel="noopener noreferrer nofollow">GitHub Releases</a>.

**Linux:**

```bash
curl -L "https://github.com/SigNoz/foundry/releases/download/v0.0.1-rc.2/foundry_linux_$(uname -m | sed 's/x86_64/amd64/g' | sed 's/aarch64/arm64/g').tar.gz" -o foundry.tar.gz
tar -xzf foundry.tar.gz
```

**macOS:**

```bash
curl -L "https://github.com/SigNoz/foundry/releases/download/v0.0.1-rc.2/foundry_darwin_$(uname -m | sed 's/x86_64/amd64/g' | sed 's/arm64/arm64/g').tar.gz" -o foundry.tar.gz
tar -xzf foundry.tar.gz
```

Run `foundryctl` from the unpacked directory (e.g. `./foundry/bin/foundryctl`) or add that directory to your `PATH`.

### Step 2. Create casting.yaml

For Docker, use `mode: docker` and `flavor: compose`.


<Admonition type="info">
For a complete guide on casting configuration, see <a href="https://github.com/SigNoz/foundry/blob/main/docs/casting.md" target="_blank" rel="noopener noreferrer nofollow">How to Write a Casting</a>.
</Admonition>

```yaml
apiVersion: v1alpha1
metadata:
  name: {deployment-name}
spec:
  deployment:
    mode: docker
    flavor: compose
```

- Replace `{deployment-name}` with your deployment name (e.g. signoz).

### Step 3. Verify dependencies

```bash
foundryctl gauge -f casting.yaml
```

### Step 4. Deploy

```bash
foundryctl cast -f casting.yaml
```

Foundry generates the Compose file and configs, then starts the containers.

### Install with Docker Compose

1. <CloneRepo />

2. To install SigNoz, enter the `docker compose up` command, specifying the following:
    - `-f` and the path to your configuration file
    - `-d` to run containers in the background
    - `--remove-orphans` to remove orphaned containers

  ```bash
  cd docker
  docker compose up -d --remove-orphans
  ```

## Install a specific version of SigNoz

<Tabs>
<TabItem value="foundry-version" label="Foundry" default>

Pin the SigNoz (and optionally ingester) image version in your `casting.yaml` by overriding the molding specs, then deploy:

```yaml
apiVersion: v1alpha1
metadata:
  name: {deployment-name}
spec:
  deployment:
    mode: docker
    flavor: compose
  signoz:
    spec:
      image: signoz/signoz:{version}
  # Optional: pin the ingester (OTel Collector) to match
  ingester:
    spec:
      image: signoz/signoz-otel-collector:{version}
```

- Replace `{deployment-name}` with your deployment name (e.g. signoz).
- Replace `{version}` with the version tag from [SigNoz releases](https://github.com/SigNoz/signoz/releases) (e.g. `0.110.0`). Use the same version for both images when pinning.

Then run:

```bash
foundryctl cast -f casting.yaml
```

</TabItem>
<TabItem value="manual-version" label="Manual install">

1. <CloneRepo />

2. Check out the version tag you want:
   ```bash
   git checkout {version}
   ```
   - Replace `{version}` with the version tag from [SigNoz releases](https://github.com/SigNoz/signoz/releases) (e.g. `v0.6.1`).

3. Start the stack with Docker Compose:
   ```bash
   cd docker
   docker compose up -d --remove-orphans
   ```

## Troubleshooting

- **Containers exit or stay unhealthy**:
  - Run `docker compose logs` (from the directory containing your Compose file) to inspect logs.
  - For Foundry, the Compose file is in the `pours/` directory after you run `foundryctl cast`.
  - Ensure at least 4GB memory is allocated to Docker.
- **Port already in use**: Ports 8080, 4317, and 4318 must be free. Stop conflicting services or change the published ports in your Compose configuration.
